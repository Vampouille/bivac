Conplicity

Automatic backups for Docker containers.

Install from source

$ go get github.com/camptocamp/conplicity

Example usage

$ conplicity \
  -u s3+http://s3-eu-west-1.amazonaws.com/<my_bucket>/<my_dir> \
  --aws-access-key-id=<my_key_id> \
  --aws-secret-key-id=<my_secret_key>

Launch with Docker

$ docker run -v /var/run/docker.sock:/var/run/docker.sock:ro  --rm -ti \
   -e CONPLICITY_TARGET_URL=s3+http://s3-eu-west-1.amazonaws.com/<my_bucket>/<my_dir> \
   -e AWS_ACCESS_KEY_ID=<my_key_id> \
   -e AWS_SECRET_ACCESS_KEY=<my_secret_key> \
     camptocamp/conplicity

Features

Conplicity allows to backup Docker volumes using the Duplicity, RClone, and Restic engines.

Built for Docker

Conplicity plugs onto the Docker socket to detect local volumes, automatically.

Prometheus Ready

Prometheus Support is built-in to easily ship metrics each time conplicity runs.

Multiple Clouds

Conplicity supports both AWS and Swift as bucket providers.

Multiple Engines

Conplicity uses Duplicity, RClone and Restic as backends to give you full control of your backup mode.
